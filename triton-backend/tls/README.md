# Local TLS with mkcert (ECDSA)

- Public CAs (for example DigiCert or Let's Encrypt) require real domain ownership.
- For local, we use `mkcert`.

## 1. Install mkcert

```bash
# macOS
brew install mkcert
```

## 2. Install local mkcert CA (one-time)

```bash
mkcert -install
```

`mkcert -install` does two things:

- Creates a local root CA (if it does not already exist).
	This is `rootCA.pem` plus its private key in mkcert's CA folder.
- Installs/trusts that root CA in local trust stores.
	This includes the system trust store (and browsers that use it).

As a result, certificates generated by mkcert are trusted locally.

## 3. Generate ECDSA server certificate and key

```bash
cd tls
mkcert -ecdsa -cert-file cert.pem -key-file key.pem localhost 127.0.0.1 ::1
```

This creates:

- `cert.pem`: server certificate for `localhost` and `127.0.0.1`
- `key.pem`: ECDSA private key (secret)

## 4. Use both files on the backend server

- Server presents `cert.pem`.
- Server proves possession of `key.pem`.

## 5. Firefox trust (if still needed)

If Firefox still does not trust your local certificate, import `rootCA.pem` from mkcert's CA folder:

```bash
mkcert -CAROOT
```

In Firefox:

- Settings -> Privacy & Security
- Search for Certificates
- Click Manage Certificates
- Authorities tab -> Import...
- Select `rootCA.pem`
- Enable: Trust this CA to identify websites
